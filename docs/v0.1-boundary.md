# v0.1 Boundary: Agent-as-Code

This repository is an executable v0.1 packet for **agent specifications as code**.

## Scope

- Represent agent roles, authority, tools, handoffs, and receipts as reviewable source.
- Keep contracts explicit so contributors can validate or reject changes without hidden assumptions.
- Provide a concrete starter pattern for other `*-as-code` repos.

## Non-canon posture

This repository is `seed`/`v0.1-draft` only until explicitly adopted by HUMMBL authority.

## Boundaries

- Do not use this repo to define universal agent standards.
- Do not publish or endorse legal, medical, or safety claims.
- Do not merge private operational details as public canon.
- Do not infer adjacent repo canon from this boundary document.
- Do not treat `base`/`human`/`founder` as runtime authority; use explicit scope fields.

## Initial practical user

Engineering and platform teams adopting agents who need:

- explicit authority and escalation boundaries,
- auditable handoff behavior,
- reproducible review-ready evidence.
